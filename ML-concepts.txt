WHAT IS MACHINE LEARNING

Machine Learning (ML) is a subset of Artificial Intelligence that enables systems to learn from data and improve performance on tasks without being explicitly programmed.

Instead of hard-coded rules, ML models discover patterns automatically with minimal human intervention.

---

CORE COMPONENTS

* Data: The fuel of ML. Quality, quantity, and diversity determine performance.
* Model: A mathematical function that maps inputs to outputs.
* Algorithm: The method used to train the model (optimize parameters).

---

TYPES OF MACHINE LEARNING

SUPERVISED LEARNING
The model is trained on labeled data.

* Goal: Learn mapping from input to output
* Most widely used in industry

Types:

* Classification
  Predict discrete class labels
  Example: Spam detection

* Regression
  Predict continuous values
  Example: House price prediction

---

UNSUPERVISED LEARNING
The model learns from unlabeled data and finds hidden patterns.

Clustering

* Groups similar data points
* Algorithms: K-Means, DBSCAN, Hierarchical Clustering, Gaussian Mixture Models
* Use cases: Customer segmentation, anomaly detection

Dimensionality Reduction

* Reduces feature space while preserving information
* Algorithms: PCA, t-SNE, UMAP, Autoencoders
* Use cases: Visualization, noise reduction, feature extraction

Generative Models

* Learn data distribution to generate new samples
* Algorithms: GANs, VAEs, Diffusion Models
* Use cases: Image synthesis, data augmentation, anomaly detection

---

REINFORCEMENT LEARNING
An agent learns to make decisions by interacting with an environment.

* Learns through rewards and penalties
* Optimizes long-term cumulative reward

Key Components:

* Agent: Decision maker
* Environment: System the agent interacts with
* State: Current situation
* Action: Possible moves
* Reward: Feedback signal
* Policy: Strategy mapping states to actions
* Value Function: Expected future reward
* Q-Function: Expected reward for action in a state

---

SEMI-SUPERVISED LEARNING

* Uses small labeled data and large unlabeled data
* Reduces annotation cost

Techniques:

* Label Propagation
* Self-Training
* Co-Training
* Pseudo-Labeling

Applications:

* Medical imaging
* NLP
* Speech recognition

---

SELF-SUPERVISED LEARNING

* Model generates its own labels from data

Techniques:

* Masked prediction
* Contrastive learning
* Rotation prediction

Examples:

* BERT, GPT, SimCLR, DINO

---

KEY ML CONCEPTS

* Features: Input variables
* Labels: Output variable
* Training Data: Used to train model
* Validation Data: Used for tuning
* Test Data: Used for evaluation

EVALUATION METRICS

Classification

* Accuracy
* Precision
* Recall
* F1 Score
* Confusion Matrix

Regression

* Mean Absolute Error (MAE)
* Mean Squared Error (MSE)
* Root Mean Squared Error (RMSE)
* R² Score

---

BIAS-VARIANCE TRADEOFF

* Bias: Error due to overly simple model
* Variance: Error due to sensitivity to data

Goal: Balance both

---

OVERFITTING VS UNDERFITTING

* Overfitting
  Learns noise
  High training accuracy, low test accuracy

* Underfitting
  Model too simple
  Poor performance on both train and test
